# Reviewer Response

Use this file for rebuttal letters, revision strategy, and point-by-point replies.

## Tone

- Be respectful, direct, and evidence-based.
- Thank reviewers briefly, then address the substance.
- Do not sound defensive, sarcastic, or overly apologetic.
- Admit limitations when the reviewer is right.
- When disagreeing, explain the reasoning and offer a manuscript change if possible.

## Response Structure

For each comment:

1. Quote or summarize the reviewer comment.
2. State the action taken.
3. Give the rationale.
4. Provide the revised manuscript text or location.

## Response Templates

Agreement:

"We thank the reviewer for this helpful comment. We have revised the Methods to clarify [specific item]. The revised text now states: [text]."

Partial agreement:

"We agree that [point] required clarification. Although [reason the original approach remains appropriate], we have revised the manuscript to make this limitation explicit."

Disagreement:

"We respectfully agree that this is an important issue. However, [evidence or design rationale]. To avoid overinterpretation, we have revised the Discussion to state [more conservative claim]."

Additional analysis:

"In response to this comment, we conducted an additional analysis using [method]. The results were [brief result], and we have added these findings to [section/table/figure]."

Cannot perform requested analysis:

"We agree that [analysis] would provide useful information. However, [data were not collected / design does not support it / sample size is insufficient]. We have added this as a limitation and revised the conclusion to avoid implying [unsupported claim]."

## Strategy

- Prioritize major conceptual and methods comments before language edits.
- Track whether each manuscript change affects abstract, methods, results, discussion, tables, and figures.
- When multiple reviewers ask related questions, harmonize the answer so the revised manuscript tells one coherent story.
- For biomechanics comments, check whether the manuscript needs clearer processing, event definition, filtering, normalization, reliability, or modeling assumptions.
